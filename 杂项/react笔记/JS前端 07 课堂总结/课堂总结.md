## npmjs.com 怎么用

### 为什么是 npmjs.com

正经项目都会发布到代码仓库上分发，因此无论是质量、稳定性、活跃度都比从各种网站上复制粘贴的代码要好。npmjs.com 是最大的 js 代码仓库，是行业的事实标准。

以五彩纸屑（confetti）业务为例，先找包实现，再从搜索引擎上找代码：
[codepen.io/longbin/pen/BaKVemE](https://codepen.io/longbin/pen/BaKVemE)，对比其优劣。

### 通过简单的查成分判断包的质量

选包第一步，看他的维护程度。

首先文档是最重要的，文档又分为概述和 API 文档，概述会讲这个包的设计与使用方法，而 API 文档会讲这个包的每个函数的参数和返回值，这两个文档都是非常重要的，选包的时候至少有一个才能放心用。

其次是维护积极性，这个包在它的组织内是什么优先级的维护，有多少未解决的 issue，社区是否有人回答问题等等。

再就是下载量，可以得知这个包在社区的接受度如何，是一个很好的参考。

若是上面这些都没有合适的，实在没有更好的选择，就要考虑自己分支然后维护，因此自己和团队是否能维护这个包也是一个重要的考虑因素。

## 前端开发的工具箱 —— 组件库

有时候需要一个方案让团队快速写出一个设计风格统一，并且交互丰富的页面，这时候就需要用到组件库。

因为组件库一般都功能较多，并且很重交互，所以不能在不同的前端框架之间通用，对于 React，有相当多的组件库可供选用。

一些比较好用的组件库：

1. ant-design：国内组件库的标杆，服务于后台开发，功能全面，文档齐全，社区活跃，但是样式比较老套。
2. material-ui：Google 的设计语言，在国外非常流行，但是国内使用率不高，写起来比较繁琐。
3. semi-design：字节的组件库，功能全面，样式现代，很适合作一些直接面向终端用户的项目，但是维护程度不如 ant-design，有一些奇奇怪怪的小坑。

## Ant Design 快速上手

这里以 ant-design 为例，来快速上手一个组件库。

首先，一切从搜文档开始，**大部分使用组件库的场景就是反复查文档并复制粘贴，不看文档使用组件库的效率甚至比手写还低**。

第二件事，找到文档上的搜索框，这是初上手时在几十篇文档中翻找出想要的内容的最快的方法。
